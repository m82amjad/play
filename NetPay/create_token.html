Merchant
merchant.merchant_id
Merchant's unique identifier ID issued by NetPay
merchant.operation_type
Identifier of API operation for the request. Value must be a member of the following list. The values are case sensitive.
  • CREATE_TOKEN
merchant.operation_mode
Value must be a member of the following list.
  • 1 : Live
  • 2 : Test

Transaction
transaction.source
Indicates the source of this transaction. Value must be a member of the following list. The values are case sensitive.
  • INTERNET : Transaction conducted over the Internet.
  • MOTO : Transaction received by mail or telephone.

Payment Source
payment_source.type
Identifier of the payment source. Value must be a member of the following list. The values are case sensitive.
  • CARD
payment_source.card.card_type
Identifier of payer's card. Value must be a member of the following list. The values are case sensitive.
  • VISA : Visa
  • VISAUK : Visa Debit K
  • ELEC : Visa Electron
  • MCRD : MasterCard
  • MCDB : MasterCard Debit
  • MSTO : Maestro
  • AMEX : American Express
  • DINE : Diners
payment_source.card.number
Credit/Debit card number as printed on the card. Data is a string that consist of the characters 0-9.
payment_source.card.expiry_month
Expiry month as printed on the card. Data is a string that consist of the characters 0-9. E.g. 01, 02
payment_source.card.expiry_year
Expiry year as printed on the card. The Common Era year is 2000 plus this value. Data is a string that consist of the characters 0-9. E.g. 13, 14, 15
payment_source.card.holder.title
Card holder's title as printed on the card.
payment_source.card.holder.firstname
Card holder's first name as printed on the card.
payment_source.card.holder.middlename
Card holder's middle name as printed on the card.
payment_source.card.holder.lastname
Card holder's last name as printed on the card.
payment_source.card.holder.fullname
Card holder's last name as printed on the card.

Token Mode
token_mode
Identifier of the token type. Value must be a member of the following list. The values are case sensitive.
  • PERMANENT : Must be used if 'tokenization' enabled on merchant's account profile.
  • TEMPORARY : Must be used for temporary purposes. Token will be expired in 1 hour after creation.

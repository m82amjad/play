Merchant
merchant.merchant_id
Merchant's unique identifier ID issued by NetPay
merchant.operation_type
Identifier of API operation for the request. Value must be a member of the following list. The values are case sensitive.
  • AUTHORIZE
merchant.operation_mode
Value must be a member of the following list.
  • 1 : Live
  • 2 : Test

Transaction
transaction.transaction_id
Merchant's unique identifier for this transaction to distinguish it from any other transactions on the order.
transaction.amount
Payment amount for this transaction. Expressed as a decimal number in the units of the currency. E.g. 25.47 in GBP is the amount 12 Pound 47 Penny. '.' represents valid decimal number.
transaction.currency
Currency for this transaction. Expressed as an ISO 4217 alpha code. E.g. GBP, EUR, USD
transaction.reference
An optional identifier of this transaction. E.g. Invoice number.
transaction.source
Indicates the source of this transaction. Value must be a member of the following list. The values are case sensitive.
  • INTERNET : Transaction conducted over the Internet.
  • MOTO : Transaction received by mail or telephone.
transaction.description
Description of this transaction.

Payment Source
payment_source.type
Identifier of the payment source. Value must be a me mber of the following list. The values are case sensitive.
  • TOKEN
payment_source.token
Token against the saved card.
payment_source.card.security_code
Card Verification Code (CVC), as printed on the back or front of the card. Data is a string that consist of the characters 0-9.

Billing
billing.bill_to_company

billing.bill_to_address
Address of the card holder. House no/name, street.
billing.bill_to_town_city
Town or City of the card holder's address.
billing.bill_to_county
County or State of the card holder's address.
billing.bill_to_postcode
Post Code or Zip of the card holder's address.
billing.bill_to_country
ISO standard alpha country code of the card holder's address.

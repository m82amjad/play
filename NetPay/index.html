NetPay API: REST

PHP Example Code Set

This set contains working example code to help you integrate your application into the payment gateway. Below are the transaction types covered in this example.

Transaction
Purchase Transaction using Card
purchase_card.html		 It is a single transaction that authorise the payment and transfer funds from payer's account to merchant account. Purchase is a operation type which the Authorise and Capture are completed at the same time. It is a common type of payment model which used by the merchant to accept card payments. Purchase operation can be done via Debit/Credit Card or Stored Card Token.
Purchase Transaction using Token
purchase_token.html		 It is a single transaction that authorise the payment and transfer funds from payer's account to merchant account. Purchase is a operation type which the Authorise and Capture are completed at the same time. It is a common type of payment model which used by the merchant to accept card payments. Purchase operation can be done via Debit/Credit Card or Stored Card Token.
Refund Transaction
refund.html		 Request a refund for a transaction.
Retrieve Transaction
retrieve.html		 Request to retrieve the details of a transaction.
Token
Create Token
create_token.html		 Request for the gateway to store card information against a token.
Retrieve Token
retrieve_token.html		 Request to retrieve the card details saved against the specified token. The card details returned include the masked card number, card expiry, and card type.
Delete Token
delete_token.html		 Request to delete token.
3DS

3DS: Check enrollment using Token

3DS: Retrieve enrollment
retrieve_ddds.html		 Retrieves the 3DS Operation Result.
3DS: Process ACS Result
ddds_process_acs_result.html		 Interprets the authentication response returned from the card Issuer's Access Control Server (ACS) after the cardholder completes the authentication process.
Scheduled Payments

Best Practices

These examples make most transaction fields available to edit when submitting the transaction. This is intended for this example only to simplify the process. In your integration, the only fields you should ever display to the customer are those requiring the customer to enter data. All other fields should be generated in your code (Process.php in this example) and not be available to the customer.

In line with the above, you should never use HTML hidden fields to submit data to be used in the transaction request. Instead, you should specify and set these values within your application code (i.e. Process.php based on this example)

Merchant
merchant.merchant_id
Merchant's unique identifier ID issued by NetPay
merchant.operation_type
Identifier of API operation for the request. Value must be a member of the following list. The values are case sensitive.
  • CREATE_SCHEDULED_PAYMENT
merchant.operation_mode
Value must be a member of the following list.
  • 1 : Live
  • 2 : Test

Transaction
transaction.amount
Payment amount for this transaction. Expressed as a decimal number in the units of the currency. E.g. 25.47 in GBP is the amount 12 Pound 47 Penny. '.' represents valid decimal number.
transaction.currency
Currency for this transaction. Expressed as an ISO 4217 alpha code. E.g. GBP, EUR, USD
transaction.source
Indicates the source of this transaction. Value must be a member of the following list. The values are case sensitive.
  • MOTO : Transaction received by mail or telephone.
transaction.description
Description of this transaction.
transaction.reference
An optional identifier of this transaction. E.g. Invoice number.

Payment Source
payment_source.type
Identifier of the payment source. Value must be a member of the following list. The values are case sensitive.
  • CARD
payment_source.card.card_type
Identifier of payer's card. Value must be a member of the following list. The values are case sensitive.
  • VISA : Visa
  • VISAUK : Visa Debit UK
  • ELEC : Visa Electron
  • MCRD : MasterCard
  • MCDB : MasterCard Debit
  • MSTO : Maestro
  • AMEX : American Express
  • DINE : Diners
payment_source.card.number
Credit/Debit card number as printed on the card. Data is a string that consist of the characters 0-9.
payment_source.card.expiry_month
Expiry month as printed on the card. Data is a string that consist of the characters 0-9. E.g. 01, 02
payment_source.card.expiry_year
Expiry year as printed on the card. The Common Era year is 2000 plus this value. Data is a string that consist of the characters 0-9. E.g. 13, 14, 15
payment_source.card.holder.title
Card holder's title as printed on the card.
payment_source.card.holder.firstname
Card holder's first name as printed on the card.
payment_source.card.holder.middlename
Card holder's middle name as printed on the card.
payment_source.card.holder.lastname
Card holder's last name as printed on the card.
payment_source.card.holder.fullname
Card holder's last name as printed on the card.

Billing
billing.bill_to_company
Name of the company which the order is being billed.
billing.bill_to_address
Address of the card holder. House no/name, street.
billing.bill_to_town_city
Town or City of the card holder's address.
billing.bill_to_county
County or State of the card holder's address.
billing.bill_to_postcode
Post Code or Zip of the card holder's address.
billing.bill_to_country
ISO standard alpha country code of the card holder's address.

Scheduled Payment
scheduled_payment.scheduled_payment_id
An identifier of the scheduled payment.
scheduled_payment.interval.frequency
It is the frequency of the payment. If unit is DAY, frequency must be between 7 and 365. If unit is MONTH, the frequency must be between 1 and 12.
scheduled_payment.interval.unit
Identifier of the interval unit. Value must be a member of the following list. The values are case sensitive.
  • DAY : Daily Payment
  • MONTH : Monthly Payment
scheduled_payment.interval.total_occurrence
Total number of payment. If value is 9999, there is no limit for payment occurrence.
scheduled_payment.interval.start_date
Identifier of the first payment date. Start day should be set at least next day of the request day.
If the start date is the 31st, and the interval unit is MONTH, the payment date will be the last day of each month.
Date format YYYY-MM-DD. Eg. 2014-12-25.
